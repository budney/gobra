# Sub-task 34.4 — HasGenericDecl Predicate
**Parent plan:** [34-test-infrastructure.md](../34-test-infrastructure.md)
**Read first:** §Scope, §Deliverables, §Verification Specifications (C9) in [34-test-infrastructure.md](../34-test-infrastructure.md); and [DECISIONS.md](../DECISIONS.md) for cross-cutting architectural constraints
**Requires:** Sub-task 34.3 ([34.3-skip-list-support-sentinel-output.md](./34.3-skip-list-support-sentinel-output.md))

## Scope (this sub-task only)
Provide the pure AST predicate that plan 35 uses to know which regression tests exercise generics, replacing the superseded grep-based approach. This sub-task covers only the deliverables listed below; other sub-tasks of plan 34 cover the remaining scope of the parent plan.

## Deliverables
- Implement `HasGenericDecl(f *ast.File) bool` in `internal/testing/runner.go`, plus the ghost predicates `funcDeclIsGeneric`, `genDeclHasGenericSpec`, and `typeSpecIsGeneric` it depends on.
- Write the Gobra postcondition specified in parent plan 34's §Verification Specifications (C9), item 1.

For full context see parent plan [34-test-infrastructure.md](../34-test-infrastructure.md) §Deliverables.

## C9
Implement all Gobra annotations from parent plan §Verification Specifications (C9) in [34-test-infrastructure.md](../34-test-infrastructure.md) whose functions/types are defined in this sub-task's deliverables — in particular the `HasGenericDecl` postcondition (item 1). Per the Active Global Constraint in [scratchpad.md](../scratchpad.md), `HasGenericDecl` and its ghost predicates are owned exclusively by this sub-task; no other plan may define a competing generic-detection function.

## Done when
- Verify `HasGenericDecl` returns `true` iff the file contains at least one generic function or type declaration, on files with and without generic declarations.
- Confirm the postcondition is present and verifies cleanly — this spec is required to pass before plan 37's self-hosting cut-over.
