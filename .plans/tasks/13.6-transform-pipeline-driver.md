**Parent plan:** [13-internal-transforms.md](../13-internal-transforms.md)
**Read first:** §Scope, §Deliverables, §Verification Specifications (C9) in [13-internal-transforms.md](../13-internal-transforms.md); and [DECISIONS.md](../DECISIONS.md) for cross-cutting architectural constraints
**Requires:** Sub-task 13.5 ([13.5-call-graph-annotations.md](./13.5-call-graph-annotations.md))

## Scope (this sub-task only)
Implement the `Apply` entry point that chains all five transforms in the required fixed order. This sub-task covers only the deliverables listed below; other sub-tasks of plan 13 cover the remaining scope of the parent plan.

## Deliverables
- Create `internal/transform/pipeline.go` with `Apply(prog *internal.Program, cfg Config) (*internal.Program, []Diagnostic)`.
- Run transforms in order: constant propagation → call graph edges → overflow → termination. All four always run; diagnostics accumulate rather than short-circuit.

For full context see parent plan [13-internal-transforms.md](../13-internal-transforms.md) §Deliverables.

## C9
Implement all Gobra annotations from parent plan §Verification Specifications (C9) in [13-internal-transforms.md](../13-internal-transforms.md) whose functions/types are defined in this sub-task's deliverables.

## Done when
- A program with overflow and termination issues produces diagnostics from both transforms in a single `Apply` call.
