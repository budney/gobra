**Parent plan:** [08-type-checker-core.md](../08-type-checker-core.md)
**Read first:** §Scope, §Deliverables, §Verification Specifications (C9) in [08-type-checker-core.md](../08-type-checker-core.md); and [DECISIONS.md](../DECISIONS.md) for cross-cutting architectural constraints
**Requires:** Sub-task 08.4 ([08.4-core-type-inference-on-ast.md](./08.4-core-type-inference-on-ast.md))

## Scope (this sub-task only)
Define the ghost type hierarchy and the two-pass stub-resolution algorithm used by the ghost type checker. This sub-task covers only the deliverables listed below; other sub-tasks of plan 08 cover the remaining scope of the parent plan.

## Deliverables
- Create `internal/info/ghosttypes.go` with `GhostType` interface, all concrete implementations (`SeqType`, `SetType`, `MSetType`, `DictType`, `OptionType`, `ADTType`, `PermissionType`, `WandType`, `StubType`), `ADTConstructor` struct, and `GhostTypeInfo` struct.
- Implement Pass 1 (register stubs) and Pass 2 (resolve bodies); assert no stubs remain after Pass 2.

For full context see parent plan [08-type-checker-core.md](../08-type-checker-core.md) §Deliverables.

## C9
Implement all Gobra annotations from parent plan §Verification Specifications (C9) in [08-type-checker-core.md](../08-type-checker-core.md) whose functions/types are defined in this sub-task's deliverables.

## Done when
- All ghost types resolve cleanly on a file containing ADT, sequence, and permission expressions.
